LON Meldpunt — GDPR/AVG-beleid · Alle persoonsgegevens worden verwerkt conform AVG (EU 2016/679) en de Belgische privacywetgeving (Wet 30 juli 2018).
Verwerkingsverantwoordelijke
LON is de verwerkingsverantwoordelijke. Per school draagt de schooldirectie de operationele verantwoordelijkheid. DPO: privacy@lon.be
Ingesteld
Rolgebaseerde toegang (RBAC)
Welzijnsdossiers zijn uitsluitend zichtbaar voor directie en aangewezen zorgcoördinator per school. Personeelsleden zien enkel eigen meldingen. Alle toegangen tot vertrouwelijke dossiers worden gelogd in een audit trail.
Actief
Bewaartermijnen
Arbeidsongevallen: 10 jaar · Welzijnsdossiers: 5 jaar na afsluiting · Medische bijlagen: consolidatie + 1 jaar · EHBO: 3 jaar · Escalaties: 5 jaar · Risico-meldingen: 7 jaar
Geconfigureerd
Recht op inzage, correctie & verwijdering
Betrokkenen kunnen inzage vragen via de schooldirectie. Verzoeken worden behandeld binnen 30 dagen. Wettelijke bewaartermijnen kunnen verwijdering beperken. Klachten: www.gegevensbeschermingsautoriteit.be
Procedure actief
Anonimisering in rapportering
Statistieken en exports bevatten geen persoonsgegevens. Namen zijn enkel zichtbaar in individuele dossiers voor bevoegden. AGODI-exports worden geanonimiseerd tenzij wettelijk anders vereist.
Actief
Minderjarigen — extra bescherming (art. 8 AVG)
Leerlinggegevens vallen onder extra bescherming. Leerlingenidentificatie is beperkt tot directie en zorgcoördinator. Leerlingenongevallen worden in rapporten geanonimiseerd. Ouderlijk toestemmingsvereiste geldt buiten wettelijke grondslag.
Actief
Technische beveiliging
Opslag: SharePoint Online (EU-datacenters, ISO 27001) · Auth: MSAL / Microsoft 365 SSO · Bijlagen: AES-256 versleuteling · Verbinding: HTTPS/TLS 1.3 · Audit trail: alle toegangen gelogd
Actief
Technische architectuur — implementatiegids voor IT-beheerder LON
Frontend — dit HTML-bestand
Klaar
Volledig zelfstandig HTML-bestand. Geen buildstap vereist. Hosten als statische pagina in SharePoint Site Assets of Azure Static Web Apps.
Hosting opties: 1. SharePoint: Site Assets → Meldpunt/index.html URL: https://<tenant>.sharepoint.com/sites/LON/SiteAssets/Meldpunt/index.html 2. Azure Static Web Apps (aanbevolen — custom domein mogelijk): meldpunt.lon.be
Authenticatie — MSAL.js + Microsoft 365
Configureren
Microsoft Authentication Library voor Single Sign-On via Microsoft 365 schoolaccounts. Rollen via Azure AD-groepen per school en functie.
// MSAL configuratie (invullen na Azure AD app-registratie): const msalConfig = { auth: { clientId: "<APP_CLIENT_ID>", authority: "https://login.microsoftonline.com/<TENANT_ID>", redirectUri: "<HOSTING_URL>" } }; // Azure AD groepen voor rolbeheer: LON-Meldpunt-Admin → LON-brede directie LON-Meldpunt-Directie-CSL → Directie CS Lijsterstraat LON-Meldpunt-Directie-CSS → Directie CS Stationsstraat LON-Meldpunt-Directie-KO → Directie Kouterschool LON-Meldpunt-Directie-TE → Directie School Ter Elst LON-Meldpunt-Zorg-[school] → Zorgcoördinatoren per school LON-Meldpunt-Personeel-[school] → Personeelsleden per school
Backend — SharePoint REST API
Configureren
Meldingen opgeslagen als items in SharePoint-lijsten. Bijlagen in Document Libraries met IRM-versleuteling.
SharePoint site: https://<tenant>.sharepoint.com/sites/LON-Meldpunt Lijsten: ├── Meldingen (centrale index) ├── Arbeidsongevallen (+ Getuigen sublijst) ├── EHBO_Registraties ├── Leerlingenongevallen ├── RisicoMeldingen ├── WelzijnDossiers (extra beveiliging: alleen directie/zorg) ├── Incidenten ├── Werkopdrachten └── Notificatie_Log Document Libraries: ├── Bijlagen_Veiligheid (IRM + AES-256) ├── Bijlagen_Medisch (IRM + beperkte toegang) └── Bijlagen_Incidenten // API call — melding opslaan: const res = await fetch( `${siteUrl}/_api/web/lists/getbytitle('Arbeidsongevallen')/items`, { method:'POST', headers:{ 'Authorization':`Bearer ${token}`, 'Content-Type':'application/json;odata=verbose', 'X-RequestDigest':digest }, body: JSON.stringify({ '__metadata':{'type':'SP.Data.ArbeidsongevallenListItem'}, Title:ref, School:school, Datum:datum, Personeelslid:naam, Letsel:letsel, Status:'In behandeling' }) });

Escalatie → SMS + onmiddellijke notificatie Trigger: Item type='Escalatie' Actie: SMS via Azure Communication Services + urgent Teams bericht directie + preventieadviseur 4. Risico → werkopdracht automatisch aanmaken Trigger: Nieuw item in 'RisicoMeldingen' Actie: Item aanmaken in 'Werkopdrachten' + toewijzen 5. Welzijn → beperkte vertrouwelijke notificatie Trigger: Item in 'WelzijnDossiers' Actie: Enkel naar directie + aangewezen zorgcoördinator 6. GDPR-bewaartermijnen → automatisch archiveren Trigger: Wekelijkse scheduler Actie: Verlopen termijnen detecteren → archiveren / verwijderen
